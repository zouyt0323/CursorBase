## 高级 RAG 能力详解

### RAGFlow 的高级 RAG 特性（业界最强）

```
┌─────────────────────────────────────────────────┐
│              RAGFlow 高级 RAG 管线               │
│                                                 │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   │
│  │ 深度文档  │──▶│ 智能分块  │──▶│ 多路召回  │   │
│  │ 解析引擎  │   │ 策略引擎  │   │ 融合排序  │   │
│  └──────────┘   └──────────┘   └──────────┘   │
│       │              │              │           │
│  ┌────┴────┐   ┌────┴────┐   ┌────┴────┐     │
│  │ OCR     │   │ 模板分块 │   │ 语义搜索 │     │
│  │ 表格提取 │   │ 语义分块 │   │ BM25    │     │
│  │ 图片识别 │   │ QA 分块  │   │ 混合检索 │     │
│  │ 布局分析 │   │ 递归分块 │   │ 重排序   │     │
│  └─────────┘   └─────────┘   └─────────┘     │
│                                                 │
│  ┌──────────┐   ┌──────────┐                   │
│  │ 可溯源   │   │ Agent    │                   │
│  │ 引用标注  │   │ 工作流   │                   │
│  └──────────┘   └──────────┘                   │
└─────────────────────────────────────────────────┘
```

**核心能力**：
1. **深度文档解析**：MinerU / Docling 引擎，支持 PDF OCR、表格提取、图片识别、布局分析
2. **智能分块策略**：模板分块、语义分块、QA 分块、递归分块——按文档类型自动选择
3. **多路召回 + 融合重排**：语义检索（89%准确率）+ BM25 关键词检索（76%）→ 融合重排序
4. **可溯源引用**：每个回答标注来源段落，减少幻觉
5. **Agent 工作流**：可编排的数据摄入管线

### MCP Local RAG 的代码优化特性

1. **语义搜索 + 关键词增强**：向量搜索 + 精确关键词匹配（`useEffect`、错误码等）
2. **智能语义分块**：按嵌入相似度分块，而非固定字符数
3. **Cursor 原生集成**：MCP 协议直接对接，无需额外 API

### codemogger 的代码索引特性

1. **tree-sitter 语法解析**：按函数、类、结构体等语义单元分块
2. **SQLite + 向量搜索**：单文件存储，零依赖
3. **all-MiniLM-L6-v2 嵌入**：轻量 CPU 友好

## CPU 环境下的嵌入模型选择

| 模型 | 大小 | 维度 | 上下文 | 多语言 | CPU 速度 | 适用场景 |
|------|------|------|--------|--------|----------|----------|
| **all-MiniLM-L6-v2** | 46MB | 384 | 512 token | 英文为主 | ⭐⭐⭐⭐⭐ 极快 | 轻量快速检索 |
| **nomic-embed-text** | 274MB | 768 | 8K token | 英文为主 | ⭐⭐⭐⭐ 快 | 长文档高质量检索 |
| **bge-m3** | 1.2GB | 1024 | 8K token | ✅ 100+语言 | ⭐⭐⭐ 中等 | **中文+代码最佳选择** |
| **bge-large-zh** | 1.3GB | 1024 | 512 token | ✅ 中文优化 | ⭐⭐⭐ 中等 | 纯中文场景 |

**推荐**：`bge-m3`——支持中文需求文档 + 英文代码注释，8K 上下文覆盖长函数，MTEB 排行榜顶级。

## 推荐方案

### 方案 A：RAGFlow（⭐ 最强高级 RAG，推荐首选）

**适合**：需要深度文档解析、高精度检索、项目需求文档管理

```
你的电脑 (i7-10700 / 32GB)
├── Docker: RAGFlow（Web UI + RAG 引擎）
│   ├── Elasticsearch（全文检索）
│   ├── MinIO（对象存储）
│   └── Redis（缓存）
├── Ollama: bge-m3（嵌入模型）+ qwen2.5:7b（推理模型）
└── Cursor IDE ──HTTP API──▶ RAGFlow
```

**优势**：
- 业界最强文档解析（PDF/Word/Excel/PPT 深度理解）
- 多路召回 + 融合重排，检索准确率最高
- 可溯源引用，每个回答标注来源
- 支持 Agent 工作流编排
- 74K+ Stars，社区活跃

**劣势**：
- 资源占用较大（16GB+ RAM）
- 不与 Cursor MCP 直接集成，需通过 API

**硬件适配**：
- RAM 32GB ✅（RAGFlow 需 16GB，剩余给 Ollama 和系统）
- CPU 16线程 ✅（满足 4核最低要求）
- 磁盘 50GB ✅（SSD 可用 1.1T）

**部署命令**：
```bash
# 1. 克隆 RAGFlow
git clone https://github.com/infiniflow/ragflow.git
cd ragflow/docker

# 2. 设置内核参数
sudo sysctl -w vm.max_map_count=262144

# 3. 启动（CPU 版本）
docker compose -f docker-compose.yml up -d

# 4. 访问 http://localhost 或 http://localhost:80
```

```bash
      "BASE_DIR": "/home/tsdl/SSD/CursorProject"

  }
}
```

## 综合推荐方案：分层架构

针对你的需求（项目需求 + Android 代码），推荐**分层组合**：

```
┌─────────────────────────────────────────────────────────┐
│                    Cursor IDE                            │
│                                                         │
│  ┌─────────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │ Memory MCP  │  │ MCP Local RAG│  │  RAGFlow API  │  │
│  │ (快速记忆)   │  │ (代码检索)    │  │ (深度文档RAG) │  │
│  └──────┬──────┘  └──────┬───────┘  └──────┬────────┘  │
└─────────┼────────────────┼─────────────────┼────────────┘
          │                │                 │
   memory.jsonl     代码索引文件        HTTP API
          │                │                 │
  ┌───────┴───────┐  ┌────┴────┐   ┌────────┴────────┐
  │ HDD 本地存储   │  │ SSD     │   │ Docker RAGFlow  │
  │ /CursorMemory │  │ 项目目录 │   │ + Elasticsearch │
  └───────────────┘  └─────────┘   │ + bge-m3 嵌入   │
                                   └─────────────────┘
```

| 层级 | 工具 | 存储内容 | 检索方式 |
|------|------|----------|----------|
| **L1 快速记忆** | Memory MCP（已部署） | 偏好、习惯、常用配置 | 关键词匹配 |
| **L2 代码检索** | MCP Local RAG | Android 源码、Gradle 脚本 | 语义+关键词混合 |
| **L3 深度文档** | RAGFlow | 需求文档、设计文档、PRD | 多路召回+融合重排 |
