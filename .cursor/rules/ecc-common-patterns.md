# 通用设计模式

## 骨架项目

实现新功能时：
1. 搜索成熟的骨架项目/模板
2. 评估安全性、可扩展性、适用性
3. 以最佳匹配为基础开发

## API 响应格式

统一使用结构化响应：

```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "meta": { "total": 100, "page": 1, "limit": 20 }
}
```

## 仓库模式

将数据访问逻辑封装在仓库层：
- 业务逻辑不直接操作数据库
- 仓库提供统一的 CRUD 接口
- 便于测试（可 mock 仓库层）

## 依赖注入

- 通过构造函数注入依赖
- 面向接口编程，不依赖具体实现
- 提高可测试性和可替换性
