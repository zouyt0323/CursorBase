# 测试规范

## 最低测试覆盖率：80%

测试类型（均为必需）：
1. **单元测试** — 独立函数、工具类、组件
2. **集成测试** — API 端点、数据库操作
3. **端到端测试** — 关键用户流程

## 测试驱动开发（TDD）

必须遵循的工作流：
1. 编写失败的测试（红）
2. 编写最少代码使测试通过（绿）
3. 重构优化（重构）

## 测试最佳实践

- 测试名称描述预期行为（如 `test_用户注册_邮箱无效_应返回错误`）
- 每个测试只验证一个行为
- 使用 mock 隔离外部依赖
- 测试边界条件和异常情况
- 测试文件与源文件同目录或镜像目录结构

## 自动化测试

- CI/CD 中自动运行所有测试
- 测试失败时阻止合并
- 定期检查测试覆盖率趋势
