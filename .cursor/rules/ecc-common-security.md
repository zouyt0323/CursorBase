# 安全规范

## 提交前必检清单

- [ ] 无硬编码密钥（API Key、密码、Token）
- [ ] 所有用户输入已验证
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护（HTML 转义）
- [ ] CSRF 防护已启用
- [ ] 认证/授权已验证

## 密钥管理

- 使用环境变量或密钥管理服务
- 永远不要将密钥提交到代码仓库
- 使用 `.env` 文件 + `.gitignore`
- 定期轮换密钥

## 安全事件响应

发现安全漏洞时：
1. 立即评估影响范围
2. 修复漏洞
3. 轮换可能泄露的密钥
4. 记录事件和修复措施

## 依赖安全

- 定期更新依赖版本
- 使用安全扫描工具检查已知漏洞
- 锁定依赖版本（lock 文件）
