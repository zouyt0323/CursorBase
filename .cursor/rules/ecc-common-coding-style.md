# 通用编码风格

## 不可变性（关键原则）

始终创建新对象，不要修改已有对象：

```
// 伪代码
错误：modify(original, field, value) → 原地修改
正确：update(original, field, value) → 返回新副本
```

不可变数据可防止隐藏副作用，简化调试，支持安全并发。

## 文件组织

- 每个文件不超过 500 行
- 每个函数不超过 50 行
- 相关功能放在同一目录下
- 公共接口与内部实现分离

## 错误处理

- 捕获具体异常，不要捕获通用 `Exception`
- 错误消息包含上下文信息（什么操作、什么输入、为什么失败）
- 使用自定义异常类区分不同错误类型
- 资源清理使用 try-finally 或上下文管理器

## 输入验证

- 在系统边界验证所有外部输入
- 使用白名单而非黑名单
- 验证数据类型、长度、格式、范围
