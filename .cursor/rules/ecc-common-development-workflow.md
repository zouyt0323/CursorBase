# 开发工作流

## 功能实现流程

1. **先计划**
   - 分析需求，识别依赖和风险
   - 分解为可管理的阶段

2. **测试驱动开发（TDD）**
   - 先写失败的测试
   - 编写最少代码使测试通过
   - 重构优化

3. **代码审查**
   - 自审：检查逻辑、安全、性能
   - 确保所有测试通过
   - 检查是否有硬编码密钥

4. **提交与合并**
   - 遵循 Git 提交规范
   - 小步提交，每次提交有明确目的
