# 性能优化

## 上下文管理

- 只加载当前任务需要的文件
- 大文件使用搜索而非全量读取
- 避免在上下文中放入不相关的代码

## 构建排错

构建失败时的排查步骤：
1. 读取完整错误信息
2. 检查依赖版本是否匹配
3. 清理缓存后重试
4. 搜索类似错误的解决方案

## 缓存策略

- 频繁访问的数据使用缓存
- 设置合理的过期时间
- 缓存失效时有降级方案

## 数据库优化

- 为常用查询添加索引
- 避免 N+1 查询问题
- 大数据量使用分页
- 只查询需要的字段
