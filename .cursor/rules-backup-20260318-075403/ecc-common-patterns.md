# 通用模式

## 骨架项目

实现新功能时：
1. 搜索经过验证的骨架项目
2. 评估选项：安全性、可扩展性、相关性、实现难度
3. 克隆最佳匹配作为基础
4. 在已验证结构内迭代

## 设计模式

### 仓库模式

用统一接口封装数据访问：
- 定义标准操作：findAll、findById、create、update、delete
- 具体实现处理存储细节（数据库、API、文件等）
- 业务逻辑依赖抽象接口，而非存储实现
- 便于切换数据源，简化模拟测试

### API 响应格式

所有 API 使用统一响应结构：
- 包含 success/status 标识
- 包含 data 载荷（错误时可为空）
- 包含 error 信息字段（成功时可为空）
- 分页响应包含元数据（total、page、limit）
