# 开发流程

本规则与 Git 工作流规则配合，描述从规划到提交的完整功能开发流程。

## 功能实现流程

1. **先规划**
   - 制定实现计划
   - 识别依赖与风险
   - 拆分为多个阶段

2. **TDD 方式**
   - 先写测试（RED）
   - 实现以通过测试（GREEN）
   - 重构（IMPROVE）
   - 确保覆盖率 ≥80%

3. **代码审查**
   - 编写代码后立即进行审查
   - 优先处理严重和高优先级问题
   - 尽量修复中优先级问题

4. **提交与推送**
   - 使用清晰的提交信息
   - 遵循约定式提交（Conventional Commits）格式
   - 详见 Git 工作流规则
