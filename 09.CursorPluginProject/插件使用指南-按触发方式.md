# Cursor 已安装插件使用指南（按触发方式）

> 按**主动触发**与**被动触发**分类，便于快速查找使用方式

---

## 一、被动触发（自动运行，无需操作）

以下插件在满足条件时自动生效，无需用户主动调用。

### 1. 语言服务与智能提示

| 插件 | 触发条件 | 效果 |
|------|----------|------|
| **Python** (ms-python.python) | 打开 `.py` 文件 | 自动补全、跳转、诊断、类型提示 |
| **Java** (redhat.java) | 打开 `.java` 文件 | 自动补全、重构、诊断 |
| **Kotlin** (mathiasfrohlich.kotlin) | 打开 `.kt` 文件 | 语法高亮、补全、诊断 |
| **clangd** (vscode-clangd) | 打开 `.c`/`.cpp`/`.h` | 补全、跳转、诊断 |
| **cursorpyright** (Cursor 内置) | 打开 Python 文件 | 类型检查、类型提示 |

### 2. 错误与质量提示（实时显示）

| 插件 | 触发条件 | 效果 |
|------|----------|------|
| **Error Lens** | 文件有错误/警告时 | 在代码行末**行内**显示错误/警告文案，无需悬停 |
| **ESLint** | 打开 JS/TS 文件 | 自动检查并显示波浪线，可配置保存时自动修复 |
| **Markdownlint** | 打开 `.md` 文件 | 自动检查 Markdown 规范，显示问题 |
| **HTML Hint** | 打开 `.html` 文件 | 自动检查 HTML 语法与规范 |
| **Code Spell Checker** | 打开任意文本文件 | 拼写错误下划线，右键可添加词库 |

### 3. 编辑时自动行为

| 插件 | 触发条件 | 效果 |
|------|----------|------|
| **Auto Rename Tag** | 修改 HTML 开始/结束标签 | 自动同步修改配对的另一侧标签 |
| **HTML CSS Support** | 在 HTML 中写 `class=` | 自动提示项目中的 CSS 类名 |
| **Better Comments** | 写注释时 | 根据 `// TODO`、`// FIXME`、`// *` 等自动高亮不同颜色 |
| **Mermaid 语法高亮** | 在 Markdown 的 ```mermaid 块内 | 自动语法高亮 |

### 4. 预览与渲染（打开/切换时）

| 插件 | 触发条件 | 效果 |
|------|----------|------|
| **Markdown Mermaid** | 打开 Markdown 预览（`Ctrl+Shift+V`） | 自动渲染 ```mermaid 图表 |
| **Mermaid Preview** | 打开 `.mmd` 或 Mermaid 预览 | 渲染 Mermaid 图表 |
| **vscode-office** | 打开 `.docx`/`.xlsx`/`.pptx` | 在编辑器内预览 Office 文件 |
| **PDF** (tomoki1207 / mathematic) | 打开 `.pdf` | 在编辑器内预览 PDF |

### 5. Git 信息（悬停/浏览时）

| 插件 | 触发条件 | 效果 |
|------|----------|------|
| **GitLens** | 悬停代码行 / 查看文件 | 自动显示该行最后修改者、时间、提交信息（blame） |

### 6. 用量监控

| 插件 | 触发条件 | 效果 |
|------|----------|------|
| **Cursor Usage** | 使用 Cursor AI 时 | 状态栏/面板自动显示 Included Requests 用量 |

## 二、主动触发（需用户操作）

以下插件需通过命令、快捷键、右键或侧边栏主动调用。

### 1. 命令面板（Ctrl+Shift+P / Cmd+Shift+P）

| 插件 | 常用命令 | 说明 |
|------|----------|------|
| **Markdown PDF** | `Markdown PDF: Export (pdf)` | 将当前 Markdown 导出为 PDF/HTML/PNG |
| **Mermaid Editor** | `Mermaid: Open Editor` | 打开 Mermaid 可视化编辑器 |
| **Mermaid Chart** | `Mermaid Chart: ...` | 连接 Mermaid Chart 云服务 |
| **Git History** | `Git: View File History` | 查看文件提交历史 |
| **GitLens** | `GitLens: Show Line History` 等 | 查看行历史、比较、仓库图 |
| **Todo Tree** | `Todo Tree: Focus on Todo Tree View` | 聚焦到 TODO 树视图 |
| **Docker** | `Docker: ...` | 构建镜像、运行容器、查看日志等 |
| **SQLite Viewer** | `SQLite: Open Database` | 打开并浏览 SQLite 数据库 |
| **Prettier** | `Format Document` 或 `Prettier: Format` | 格式化当前文件 |
| **ESLint** | `ESLint: Fix all auto-fixable Problems` | 一键修复可自动修复的问题 |

### 2. 调试（F5 / 运行配置）

| 插件 | 触发方式 | 说明 |
|------|----------|------|
| **Python Debugger** | F5 或 运行 → 启动调试 | 断点、变量、调用栈 |
| **Java Debugger** | F5 或 运行 → 启动调试 | Java 断点调试 |
| **Debugpy** | Python 调试后端 | 由 Python 插件调用 |

### 3. 侧边栏 / 资源管理器

| 插件 | 位置 | 说明 |
|------|------|------|
| **Todo Tree** | 左侧活动栏 | 树形展示所有 TODO/FIXME/XXX 等 |
| **Docker** | 左侧活动栏 | 镜像、容器、网络、卷管理 |
| **GitLens** | 源代码管理、文件/行内 | 提交图、比较、blame |
| **Git History** | 通过命令打开 | 文件历史视图 |

### 4. 右键菜单

| 插件 | 右键项 | 说明 |
|------|--------|------|
| **Git History** | `Git: View File History` | 在文件上右键 |
| **GitLens** | `Open Line History`、`Compare with...` | 在行号/代码上右键 |
| **Docker** | `Build Image`、`Run` 等 | 在 Dockerfile 上右键 |
| **Code Spell Checker** | `Add to Dictionary`、`Spelling` | 在拼写错误上右键 |
| **Prettier** | `Format Document` | 在编辑区右键 |

### 5. 快捷键（可自定义）

| 插件 | 默认快捷键 | 说明 |
|------|------------|------|
| **Format Document** | `Shift+Alt+F` (Win) / `Shift+Option+F` (Mac) | Prettier/默认格式化 |
| **Markdown 预览** | `Ctrl+Shift+V` / `Cmd+Shift+V` | 预览 Markdown（含 Mermaid） |
| **启动调试** | `F5` | 启动调试会话 |

## 四、常用配置建议

```json
// settings.json 示例
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "[javascript]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },
  "[typescript]": { "editor.defaultFormatter": "esbenp.prettier-vscode" },
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "errorLens.enabled": true,
  "todo-tree.general.tags": ["TODO", "FIXME", "XXX", "HACK"]
}
```

保存时自动格式化 + ESLint 自动修复，Error Lens 与 Todo Tree 按需开启。
